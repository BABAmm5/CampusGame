# King Game Prototype

基于 React + Vite + TypeScript + Tailwind CSS 的手机优先 H5 回合制策略游戏原型。

## 已实现内容

- 首页、开局设置页、游戏主页面
- 2 到 4 阵营的本地热座模式
- 四个阵营使用固定名称：统治者、反抗者、守护者、入侵者
- 阵营支持不对称初始属性
- localStorage 自动保存与继续游戏
- 规则说明弹窗
- 设置页规则编辑器
- 配置码导出 / 导入
- 四阵营状态卡展示：血量、金币、民、兵、武器等级、盔甲
- 阵营状态折叠、战报展开收起、底部固定操作区
- 简化规则模块化实现：
  - `types.ts`
  - `config.ts`
  - `configCodec.ts`
  - `initGame.ts`
  - `economyEngine.ts`
  - `battleEngine.ts`
  - `roundEngine.ts`
  - `victoryEngine.ts`
- 每回合自动结算经济并要求分配新增 5 单位
- 民兵转换、武器升级、盔甲强化、基础战斗结算
- 阵营死亡后的资产继承
- 战报日志持续记录关键操作

## 游戏玩法

### 目标

击败其他所有阵营，成为最后存活的阵营。

### 开局

- 选择 2 到 4 个阵营开始游戏。
- 每个阵营拥有初始血量、金币、平民、士兵、武器等级和盔甲值。
- 当前默认开局为不对称阵营：
- 阵营 1：统治者，10 血、300 金、10 民、5 兵
- 阵营 2：反抗者，8 血、200 金、5 民、8 兵
- 阵营 3：守护者，6 血、300 金、10 民、0 兵
- 阵营 4：入侵者，8 血、200 金、0 民、10 兵
- 这些参数可以在设置页直接编辑，也可以通过配置码导入。

### 回合流程

每个阵营轮到自己时，按下面顺序进行：

1. 自动结算经济
- 平民按规则产出金币。
- 士兵按规则消耗金币。

2. 分配本轮增员
- 每轮会新增固定数量单位。
- 你需要在“民”和“兵”之间分配这些新增单位。
- 未完成分配前不能结束回合。

3. 执行操作
- 民转兵：消耗一定数量平民，补充士兵。
- 兵转民：遣返部分士兵，补充平民。
- 升级武器：消耗金币，提高进攻伤害。
- 强化盔甲：消耗金币，减少受到的伤害。
- 发起进攻：选择目标阵营并派出士兵作战。

4. 结束回合
- 当前阵营操作完成后结束回合。
- 系统切换到下一个存活阵营。

### 战斗规则

- 进攻伤害由出兵数量、基础攻击和武器等级共同决定。
- 防守方盔甲会减少受到的伤害。
- 防守方会反击，进攻方可能损失士兵。
- 当防守方血量降到 0 时，该阵营灭亡。
- 灭亡阵营的金币、平民和士兵会被胜利方继承。

### 配置码

- 配置码是一段字符串，用来保存当前阵营数量和整套规则参数。
- 你可以在设置页导出配置码，复制给其他人使用。
- 也可以导入配置码，快速恢复同一套规则。
- 当前配置码只保存规则，不保存正在进行中的对局。

### 自动存档

- 游戏状态会自动保存到浏览器 `localStorage`。
- 重新打开页面后，可以在首页点击“继续游戏”。
- 如果清理浏览器站点数据，存档会被删除。

## 本地启动

```bash
npm install
npm run dev
```

默认开发地址通常为 `http://localhost:5173`。

## 打包部署

```bash
npm run build
npm run preview
```

打包产物位于 `dist/`，为纯静态文件。

## 静态托管说明

- 已设置 Vite `base: "./"`，构建结果使用相对路径。
- `dist/` 可直接部署到 Gitee Pages、GitHub Pages 或任意静态文件服务器。
- 如果你使用仓库子路径部署，也不需要额外后端支持。
